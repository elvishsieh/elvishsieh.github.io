# What is Variable?
> A variable in Python is a symbolic name that acts as a reference or pointer to an object (value) stored in the computer's memory. Unlike some other programming languages, Python is dynamically typed, meaning you do not need to declare the variable's data type beforehand; it is automatically determined when you assign a value to it
## Characteristics of Python Variables
- No Declaration Required: Unlike many other programming languages (like Java or C), you do not need to explicitly declare the type or even the existence of a variable before using it. A variable is created the moment you first assign a value to it using the assignment operator (=).
- Dynamically Typed: Python is a dynamically typed language. This means you do not have to specify the data type of a variable, and the same variable can be assigned values of different types during the program's execution.
- Object Reference: In Python, every variable is essentially a symbolic name, or label, that holds a reference (a pointer) to an object stored in memory. When you assign a value (e.g., x = 10), the variable x points to the memory location of the integer object 10.
- No Fixed Size for Integers: Unlike some other languages, Python integers have arbitrary precision, meaning their size is limited only by the available memory of the computer, not a fixed number of bits.
- Scope: Variables have a defined scope (global, local, or nonlocal) that determines where they can be accessed within a program. 

## Naming Rules and Conventions
> To ensure code is readable and functional, variable names must follow specific rules: 
- Must start with a letter (A-z) or an underscore (_).
- Cannot start with a number.
- Can only contain alpha-numeric characters and underscores (A-z, 0-9, and _).
- Are case-sensitive (age, Age, and AGE are treated as three different variables).
- Cannot be Python keywords (reserved words like if, else, for, etc.).

# What is Data Type?
> A data type is a classification that dictates what a variable or object can hold in computer programming and lets a computer know how to interpret the data's value.

# What is Functin and Method in Python?
> In Python, a function is an independent, standalone block of code, while a method is a function that is associated with an object and defined within a class. 

## What is Function?
> In Python, a function is an independent, standalone block of code.

> A function is a reusable block of code that performs a specific task and operates on data passed to it explicitly as arguments. 

- Definition: Defined using the def keyword outside of a class.
- Calling Syntax: Called directly by its name, e.g., my_function(argument).
- Independence: Functions are independent and do not require an object or class instance to be called.
- Examples: Built-in functions like len(), print(), and sum(), or any user-defined function not within a class. 

> An example code of function in Python
```python
# Example of a function
def add(a, b): # parameter in define function
    return a + b
if __name__=="__main__":
    result = add(5, 3) # argument in function call
    print("The result is: ", result) # Output: 8
```
## What is Method in Python?
> In Python,  a method is a function that is associated with an object and defined within a class.

>A method is a function that belongs to a class and defines the behavior or actions an object of that class can perform. 

- Definition: Defined using the def keyword inside a class.
- Calling Syntax: Called using "dot notation" on an object or class, e.g., object_name.method_name(argument).
- Dependence: Methods are associated with the object they are called on and can access and modify the object's internal data (attributes).
- Implicit Argument: An instance method's first parameter, conventionally named self, automatically receives the instance object it's called on. 

> An example code of method in Python
```python
import jupyturtle
class draw_graph:
    def __init__(self, n, length, angle):
        self.n = 0
        self.length = 0
        self.angle = 0
        
    def polyline(n, length, angle):
        for i in range(n):
            forward(length)
            left(angle)

    def polygon(n, length):
        angle = 360 / n
        for i in range(n):
            forward(length)
            left(angle)

if __name__=="__main__":
    make_turtle(delay=0)
    draw_graph.polyline(n=2, length=20, angle=30)
    draw_graph.polygon(n=4, length=20)
```
## Conclusion
> Essentially, all methods are functions, but not all functions are methods.

## Encapsulation
- What is encapsulation?
> Encapsulation is a core principle of object-oriented programming (OOP) that involves bundling data (attributes) and methods that operate on that data into a single unit, typically a class.
- An encapsulatin's code in Python
```python
import jupyturtle
def square():
    for i in range(4):
        forward(50)
        left(90)
if __name__=="__main__":
    make_turtle()
    square()
```
> As above code, wrapping a piece of code up in a function is called encapsulation.
## Generalization
- What is generalization?
> Generalization is about extracting common patterns, behaviors, or attributes from specific instances (like classes or functions) and abstracting them into a more general, reusable structure (superclass, interface, or generic function), enabling code reuse and reducing redundancy, primarily through inheritance (OOP) or generic types, but also via libraries, modules, or common APIs, while avoiding premature generalization (speculative generality).
> A generalizatin's code in Python
```python

    for i in range(4):
        forward(length)
        left(90)
if __name__=="__main__":

```
> Adding a parameter to a function is called **generalization** because it makes the function more general with the previous code.

# Refactoring(重構)
- What is refactoring
> Refactoring is the process of restructuring existing computer code—changing its internal structure—without altering its external behavior or functionality, primarily to improve readability, maintainability, and reduce complexity. It's like tidying up a messy room; the room still functions the same, but it's easier to find things and move around. Developers use it to clean up "technical debt," making the code easier to understand, extend, and less prone to bugs in the future. 

## Key aspects of refactoring

- Behavior preservation
    > The core principle is that the code must still do the exact same things from a user's perspective after refactoring.
- Internal improvements
    > Focuses on non-functional aspects like design, structure, and clarity.
- Small, incremental changes
    > Often done in small, safe steps (e.g., renaming a variable, extracting a method) rather than large rewrites, often guided by good unit tests to ensure nothing breaks. 

## Why developers refactor
- Improve readability
    > Makes code easier for other developers (or future you) to understand.
- Enhance maintainability
    > Easier to fix bugs or add new features later.
- Reduce complexity
    > Breaks down large, confusing parts into smaller, manageable pieces (like extracting long methods).
- Increase performance
    > Can lead to more efficient code (though this isn't the primary goal).
- Prepare for new features
    > Often done before adding new functionality to make the addition cleaner.

# Recursion(遞迴)
- What is recursion?
> Recursion is a programming technique where **a function calls itself to solve a problem** by breaking it down into smaller, simpler subproblems.

## Key Components of a Recursive Function
Every recursive function must have two main parts to prevent infinite execution and potential program crashes (stack overflows): 

- Base Case: A condition that, when met, stops the recursion and provides a direct - - - - solution to the simplest subproblem. This acts as the termination condition.
- Recursive Case: The part of the function that calls itself with modified parameters, moving the problem closer to the base case.

> A classic example is calculating the factorial of a number (n!), which is the product of all positive integers less than or equal to n **(n! = n  x (n-1)!)**. 
```python
def factorial(n):
    # Base Case: When n is 0 or 1, the recursion stops and returns 1
    if n == 0 or n == 1:
        return 1
    # Recursive Case: The function calls itself with a smaller number (n-1)
    else:
        return n * factorial(n-1)

if __name__=="__main__":
    num = 5
    print(f"The factorial of {num} is {factorial(num)}")    # Output: 120
```
> Fibonacci Sequence
```python
def fibonacci(n):
    if n == 0:
        return 0
    elif n == 1:
        return 1
    else:
        return fibonacci(n-1) + fibonacci(n-2)
if __name__=="__main__":
    print(fibonacci(10))
```
