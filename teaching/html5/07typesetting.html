Google Fonts

  • Google Fonts 提供中、英文字型讓 Web Developer 自由、免費使用
  • 使用的方法有兩種：
    1. 網址連結：連結到相對應的字型網址來設定及使用這些字型
    2. 下載字型：透過相對位置連結方式來連結下載字型的CSS設定
  • 連結到 Google Fonts 字型網址的設定步驟：
    1. 選擇你想要的英文字體 ≫按右上角紅色的+號 ≫ 點選下方黑色的Family Selected ≫
    2. 在 header 標籤中加入 fonts 網址的連結
      3. <head>
<link href='https://fonts.googleapis.com/css?family=Oswald' rel='stylesheet' type='text/css'>
</head>
    3. 在 CSS 中加入 font-family 的設定
      4. font-family: 'Oswald', sans-serif;

設定下載的字型(1/2)

  • 檢視並複製字型的設定檔：以 cwTeX's Yen, Oswald 為例
    1. 連結到 http://fonts.googleapis.com/earlyaccess/cwtexyen.css
    2. 複製所有的 CSS 設定值及修改 src 的路徑
    @font-face {
    font-family: 'cwTeXYen';
    font-style: normal;
    src: url('fonts/cwTeX/cwTeXYen-zhonly.ttf') format('truetype');
}
@font-face {
    font-family: 'Oswald';
    font-style: normal;
    font-weight: 200;
    src: url("fonts/Oswald/Oswald-Regular.ttf");
}
body{
    font-family: 'Oswald', 'cwTeXYen', sans-serif;
    font-size: 2em;}

設定下載的字型(2/2)

CSS 中的 @ 字元(1/2)

  • @ 字元的意思是： at-rules
  • 可以將 at-rules 看成：將某個特有的 CSS 規則封裝起來，然後指定做這個規則的設定
  • CSS 標準目前有六個 @ at-rules：@import, @page, @media, @font-face, @charset, @namespace
  • 例如：@font-face 專指「設定字體」這項事情

CSS 中的 @ 字元(2/2)

  • 例如：@media 專門針對「判斷設備(media)」這件事物來做 CSS 的設定
@media screen and (max-width: 1440px) /* <= 1440px */
{
    span.sizedImage
    {
        height:135px;
        width: 174px;
    }
}    

@media screen and (min-width: 1441px) /* >=1441px */
{
    span.sizedImage
    {
        height:150px;
        width: 200px;
    }
}

At-rules in Future

  • @charset — 由 CSS 來定義字元編碼方式(charset)
  • @import — 匯入外部的 CSS
  • @namespace — 告訴 CSS 引擎在解析所有 CSS 內容時，必須將前置 XML namespace 考量進去
  • 條件式規則群:
    • @media
    • @supports:
      @supports <supports-condition> {
  <group-rule-body>
}
    • @document(延期到 Level 4 of CSS 的標準訂定)

Reference to MDN

Nested(巣狀式)的規則群 — Nested(巣狀式)指令的子集合，可以被使用在 CSS 或者條件式規則群內部

  • @media — 條件式的規則，使用 media query 所定義的準則來判斷 media，例如：print, screen, tty, tv 等 media
    • @supports — 條件式的規則，如果瀏覽器判斷到這些情況的準則時，就使用這個 CSS 設定值
    • @document — 條件式的規則，如果文件()判斷到這些情況的準則時，就使用這個 CSS 設定值
    • @page — 設定列印時的頁面及外觀
    • @font-face — 設定由外部下載的字型
    • @keyframes — 設定 CSS 檔案中關於動畫的時間軸及其他相關設定值
    • @viewport — 設定小螢幕觀看結果(目前仍然只是草案階段)
    • @counter-style — 設定項目符號的樣式 (已是建議標準階段)

    特殊字元(1/2)

    • 所謂特殊字元是：<!‐‐(與 <!-- 是否有差別) , ≫, >>, &, ☔
    • 特殊字元的寫法：
      • Character Name: &character-name ;
      • 十進制(Decimal): &#decimal-numeric;
      • 十六進制(Hex)：&#xhex-numeric ;
      • 特殊字元的範例：
      Character Unicode Name Decimal Hex
      © U+00A9 &copy; &#169; &#x00A9;
      & U+0026 &amp; &#0038; &#x0026
      ☔ U+2614 undefine &#9748; &#x2614;

    特殊字元(2/2)

    • Unicode 8.0 Character Code Charts
    • 如何查 Unicode 編號，以 Unicode 2600 為例？
      • 上面的編號 + 對應到左邊的編號
      • 例如：上面的編號是 261，對應到左邊的編號是 4，所以 Unicode 的編號是：2614(☔)
    • W3C 網站的特殊字元表
    • 特殊字元表分門別類的網站

    Windows 系統中如何輸入 Uincode?

    • 限制：僅在 Windows 作業系統中的微軟注音輸入法，其他系統請參照輸入法的說明文件
    • 輸入步驟：
      1. 將輸入法切換到微軟注音輸入法
      2. 在中文輸入模式下，輸入： ` + u + Unicode 編號
    • 例如：U+00A9 則先輸入`，然後輸入 u，最後輸入編號 00A9，就可以得到 ©
    • 注意：要使用鍵盤上方的數字鍵來阿拉伯數字，不可以使用數字鍵盤來輸入，這樣會跳出 Unicode 輸入模式

    字型(Font)的種類

    • Bitmap: 早期 Dos 作業系統使用的字型，常見的格式有：bdf，pcf，fnt，hbf 等，最大缺點就是字型會失真
    • PostScript: Adobe 公司的向量字型，字型平滑、精細
      • .ps : PostScript 字型的副檔名
      • TeX, cwTeX 所產出的就是這種字型
    • TrueType: Apple 與 Microsoft 公司所推出的字型，可調整成任何大小，而且不論大小為何都很清晰且容易閱讀。
      • .ttf : TrueType 字型的副檔名
      • .tte : 是 TrueTpe 字型，是給使用者自訂定義編碼用，就像作業系統沒有這個字時，使用者可以自行編碼及造字
    • OpenType: 具有 TrueType 字型特性外，還合併較大的基本字元集，像是「字符」與「連寫」等
    • ClearType: 推 Windows 7 時所出的新字型

    網頁字型(Web Font)的種類

    • 請先看 cwTeX Yen Fonts 的設定值
    • Embedded OpenType: 與 OpenType 字型相容的字型，主要是 Microsoft 公司所設計，要內嵌在網頁上使用的字型
      • .eot: Embedded OpenType 的副檔名
      • ttf2eot: ttf轉 eot 字型的轉檔程式
    • Web Open Font Format(WOFF): 是壓縮的 OpenType or TrueType 字型，以及加入一些 meta data 的資訊，目前由 W3C 來主導及制定標準
      • W3C's Web Fonts 的標準
      • 未來會使用 WOFF 2.0 版本的字型

    WOFF Fonts(1/2)

    • WOFF: Web Open Fonts Format
    • 目前的標準是 WOFF 1.0 版本，未來將會使用改良壓縮技術的 WOFF 2.0 版本
    • 為何要使用 WOFF 字型？因為考量網路的速度及頻寬，WOFF 可以加速字型的載入
    • WOFF 轉檔工具程式
      • sfnt2woff.exe 是將 ttf or OpenType 轉成 woff 字型的程式
      • 執行轉換的指令：sfnt2woff 字型的檔名
        • D:\新增資料夾\cwTeX>sfnt2woff cwTeXFangSong-zhonly.ttf
      • 目前 WOFF 1.0 版本的壓縮率還是不高(cwTeXFongSong 字體做轉換，約 30%)，所以建議同學未來使用 WOFF 2.0 的技術

    字體架構的分類

    • 早期印刷技術就是要設計鉛字體，然後使用這些字體來印製書籍、文件，所以，早期的字體是使用鉛字來製作
    • 鉛字體每一個字的骨架設計有分為 Serifs, Sans serif:
      • Serifs: 中文翻譯：襯線字體，有邊邊角角的字體
      • Sans serif: 中文翻譯：無襯線字體，沒有邊邊角角的字體
    • 字與字之間的排列可以分成 proportional 與 monospaced
      • Proportional: 比例的字體，字的大小有比例的分別
      • Monospaced: 無比例或者固定寬度的字體，字與字之間等寬度

    字體的設計樣式

    • 羅馬字體的設計樣式
      • Serif:最具代表的字型：Times Roman 與 Garamond
      • Sans serif: 有 Helvetica, Futura, Gill Sans, Univers, Frutiger 等字型
      • Script:最具代表的字型： Zapfino
      • Monospaced: 有 Courier, Prestige Elite, Fixedsys, Monaco 等字型
    • CJK 字體的樣式與英文字體不同
      • 明體比較像 Serif，黑體比較像 Sans-serif
      • 常用的還有：仿宋體、隸書、楷書
      • 古典的有：行書、草書
      • 更早的有：甲骨文、篆書(大篆、小篆、鳥蟲篆)等
