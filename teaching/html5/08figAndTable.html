Responsive Web Design

  • RWD(Responsive Web Design)是現在網頁設計的必要條件
  • 可是，RWD 要考慮的元件及樣式相當繁雜，比如說，在大小不同的螢幕或設備要如何呈現字型大小、版面樣式、圖片及表格大小等等問題?
  • 目前還有許多 RWD 相關的標準尚未制定完成，在撰寫網頁程式碼要小心、注意瀏覽器是否支援?
  • RWD 可以使用 @media 或者撰寫 Javascript 來處理不同設備間網頁呈現的問題
  • RICG(Responsive Issues Community Group)是針對 Responsive 這個議題而成立的組織，他們與 W3C 及 HTML Living Standard 是有資訊交流與聯繫

Media Query

  • @media 選項有：all, braille(點字), embossed(浮雕), handheld(手持式), print, projection(投影設備), screen, speech(語音), tty, and tv
    • @media screen and (max-width: 1440px)
 {
    ...
 }
  • W3C 考慮的不只是手機、平板、螢幕大小的問題，還考慮到網頁要列印、簡報時，所要呈現的結果是什麼樣子？
  • braille(點字)：無障礙網頁要達到某種程度才能取得認證
  • handheld(手持式)：Google Glasses 是最典型的手持式裝置
  • speech(語音)：透過不同 lang 的設定，可以由不同翻譯機播放出不同的語音

使用 Javascript 套件

  • 網路上有人已經寫了 Javascript 程式碼，讓我們可以先使用這些未來要制定的標籤、屬性與屬性值
  • 換言之，就是目前瀏覽器可能尚未支援這些標籤、屬性與屬性值，所以在這種過渡時期，要先使用這些 Javascript 套件來暫時填補這段空窗期
  • Modernizr 是 Javascript 的套件，之前我們有介紹過，它也提供上面所講的功能
    • Add your detects ≫ Ctrl + F ≫ 找 picture, or Emoji ≫ 按 ♁ ≫ BUILD ≫ Download
    • 下載 .js 後將 script 標籤加入到 head 標籤內，請參考 modernizr.html的原始程式碼
  • 其他功能的 Javascript 套件可以到 CDN 找

Responsive Images (1/2)

  • Responsive Image 是其中一種需要特別處理的元件之一
  • Picturefill is a responsive image polyfill，意思就是標準尚未制定完成的一種過渡時期作法
  • picturefill.js 讓網頁設計者可以先使用 picture 標籤及屬性
  • 下載及使用方法：
    • 下載：picturefill.js(v3.0.2 Stable) 或 picturefill.min.js
    • 使用：在 <head> 標籤中加入
      • <head>
  <script>
    // Picture element HTML5 shiv
    document.createElement( "picture" );
  </script>
  <script src="picturefill.min.js" async></script>
</head>
  • picturefill.js 的範例及原始程式碼

Responsive Images (2/2)

  • respimage 是另一種 responsive images polyfill
  • 下載及使用方法：
