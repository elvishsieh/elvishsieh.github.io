網際網路的歷史(1/4)

  • 在1960年初期，一些科學家及軍事界的人員，發現電腦可共享研發資訊，將帶來的可觀的價值
  • 在1962年：
    • 麻省理工學院J.C.R. Licklider首先提出全球網路的概念
    • 由國防尖端研究專案機構(DARPA)接手進行研發
    • 加州大學洛杉磯分校的Leonard Kleinrock 教授發展出封包交換技術理論，為網際網路連結奠基
    • 麻省理工學院的 Lawrence Roberts 利用撥接式電話線，將麻州的電腦與加州的電腦相連
    • 同時也發現電話線的電路交換技術不敷運用，Kleinrock 的封包交換理論受到肯定

網際網路的歷史(2/4)

  • 於1969年麻省理工學院的 Roberts 轉到國防尖端研究專案機構(DARPA)，開始發展他所籌畫的Advanced Research Projects Agency (ARPA)
  • 1970年美國各大學紛紛加入 ARPANet
  • 197~ 年由 BBN 的 Bob Kahn 所提出 TCP/IP的架構，後來由史丹福的Kahn與Vint Cerf及其他人在70年代期間開發
  • 在1978年時由貝爾實驗室發明 Unix to Unix備份通訊協定(UUCP)
  • 1979年時新聞討論群(Usenet) 根據UUCP開始啟用

網際網路的歷史(3/4)

  • 1980年時美國國防部所採納 TCP/IP，取代先前的網路控制通訊協定(NCP)
  • 1983年廣為全球使用，例如 Archive, telnet, mail, ftp
  • 1989年粒子物理學歐洲實驗室(CERN)的 Tim Berners Lee 研發出 World Wide Web(W3)通訊協定
  • 1991年明尼蘇達大學開發出 Gopher(小田鼠)

網際網路的歷史(4/4)

  • 1993年超電算應用國家中心(NCSA)的 Marc Andreessen 與他的小組成員開發了圖形瀏覽器Mosaic
  • 1993年之後，Andreessen 更轉往網景公司(Netscape)，開發領航員(Navigator)
  • 1998年微軟(Microsoft)將開發出Windows 98 的作業系統(O.S)與 Internet Explorer(IE)瀏覽器緊密整合
  • 之後，Netscape 公司開放原始程式碼(open source)，yield Mozilla Firefox browser

網頁、網站與網站伺服器

  • 網頁(Web Page)：由HTML 語法、CSS(cascading Style Sheet)、Javascript 語言撰寫而成的頁面
  • 網站(Web Site)：由許多網頁及一些多媒體的檔案所構成的網頁架構
  • 首頁(Home Page)：網站的第一個頁面，檔名通常是 index.html, index.htm, index.asp, index.php, index.aspx
  • 網站伺服器(Web Server)：主要是提供網頁製作者存放他們製作的網頁或網站的一種軟體，例如： IIS, Apache HTTP Server ("httpd"), nginx, lighttpd 等

Client/Server Architecture

  • 2-Tier Client / Server 架構
    • e.g. 郵政信箱 / 郵局
    • Outlook, Thunderbird / Mail Server(sendmail, postfix,qmail)
    • Browser / Web server(Apache, IIS, lighttpd)

    • Client PC / Application Server / Database Server

IP & 主機名稱(Hostname)

  • 電腦只懂得 0 與 1
  • IP Address 是給電腦看的(e.g. 163.17.131.8)
  • Hostname 是給人看的(e.g. web2.nutc.edu.tw)
  • DNS(Domain Name Server)
    • 將 Hostname 轉成 IP
    • Internet Corporation for Assigned Names and Numbers (ICANN): 管理所有的 Domain Name, e.g. .cn, .tw, .uk
    • 台灣網路資訊中心(TWNIC): .tw Domain Name
    • Domain Name(e.g. nutc.edu.tw, gov.tw, idv.tw, com.tw)
  • 特殊的 IP Address：
    • localhost: 127.0.0.1
    • 詳細說明請參考「鳥哥的 Linux 私房菜」

測試 IP & Hostname

  • 開啟 cmd.exe 程式：開始>執行>輸入 cmd
  • 輸入：nslookup www.nutc.edu.tw //問 DNS www.nutc.edu.tw 這個 Host(主機)的 ip 是什麼
  • 輸入：ping www.yahoo.com.tw // 測試 www.yahoo.com.tw 網站是否存活在網路上
  • 輸入：ping localhost
  • 輸入：nslookup localhost
  • 輸入：ipconfig /all //取得電腦ip 位址、實體網卡位址(MAC, Media Access Control)
  • 輸入：ipconfig /all | find /I "ip" //將 ipconfig /all 指令取得的訊息傳給find 指令，/I：不分字母大小寫尋找字串(string)
  • 輸入：ipconfig /all | find /I "實體" //找 MAC，Laptop 可能找出多個值

通訊協定與連接阜

  • IPv4: 4 bytes, 32bits
  • IPv6: 6 Bytes, 48bits
  • Application Layer:
    • HTTP(Hypertext Transfer Protocal), 80 port
    • HTTPS(Security), 443 port
    • FTP(File Transfer Protocal), 21 port
    • SMTP(Simple Mail Transfer Protocal), 25 port
  • Presentation Layer:
    • SSL(Secure Sockets Layer)
    • TLS(Transport Layer Security)
  • Unix /etc/services

何謂 URL?

  • URL is Uniform Resource Locator 第一個英文字母的縮寫
  • A subset(子集合) of Uniform Resource Identifier (URI)
  • URI syntax(語法)： resource_type://hostname:port/filepathname?query_string#anchor
  • Example:
    • http://www.ntu.edu.tw/
    • http://www.ntu.edu.tw/index.html
    • http://www.ntu.edu.tw:80/index.html
    • ftp://ftp.ntu.edu.tw:21/

何謂 HTML?

  • HTML 是 Hyper Text Markup Language 的縮寫

HTML 的缺點

  • HTML 標籤不能夠表示單一的意義，容易產生混淆
  • 廠商自行定義一些標籤，造成不一致性
    • IE 的 <IFFRAME> 標籤
    • Netscape 的 <LAYER>標籤
  • 不夠嚴謹的結束標籤
    • HTML <br>
    • XHTML <br> <br />

何謂 XML?

  • XML 是 eXtensible Markup Language 的縮寫
  • 可以轉換成各式各樣的資料型態：
    • Microsoft Office 2007 Version: .docx, .xlsx, pptx
    • 可轉成文字檔 e.g. text, html, rtf, odf
    • 可轉成圖形檔 e.g. svg
      • http://zh.wikipedia.org/zh-hant/File:URI_Venn_Diagram.svg
      • https://upload.wikimedia.org/wikipedia/commons/f/ff/URI_Venn_Diagram.svg
      • 原始程式碼

何謂 XHTML?

  • XHTML 是 eXtensible HTML 的縮寫
  • 為了實現 XML 的過度產品
  • 修補了 HTML 的部份缺失 e.g.<br> <br />
  • 無法實現 XML 的所有特性 e.g. svg 檔案格式
  • 自從 2010 年 W3C 核定了 HTML5 標準以後，支持XML語法的陣營退出 XHTML 標準的制定

SGML 與 DTD

  • SGML 是 Standard Generalized Markup Language 的縮寫
  • 它是 World Wide Web Consortium (W3C) 所制定的標準；HTML 是它的一個子集合
  • DTD 是 Document Type Definition 的縮寫
